  検査ロジック（A/B 共通）

  package.json の追加対象パッケージごとに npm
  registry（https://registry.npmjs.org/<pkg>）を叩いて、以下をスコアリング：

  1. ライフサイクルスクリプト … preinstall / install / postinstall の中身
    • curl|wget|nc|powershell|iwr|Invoke-WebRequest
    • eval, base64, \\x[0-9a-f]{2}, String.fromCharCode
    • process.env\.(NPM_TOKEN|AWS_|GITHUB_TOKEN|SSH_|SECRET|API_KEY)
  2. 新規 / リリース新鮮度：直近公開（< 14 日）は赤
  3. メンテナ数：1 人 + 新しい = 警戒度上げ
  4. DL 数（https://api.npmjs.org/downloads/point/last-week/<pkg>）：極端に少ないのに人気名
     に酷似 → タイポスクワット疑い
  5. タイポスクワット：人気パッケージリスト（同梱）と Levenshtein 距離 ≤ 2
  6. 既知の侵害リスト（GHSA (https://github.com/advisories) / Socket (https://socket.dev/)
     などのフィードを定期取得）と一致するか
必要であれば、OSS の safenpm (https://www.safenpm.dev/)のコードも活用(取り込む)
  Hook での挙動（A）

  • マッチャ: ^(npm|pnpm|yarn)\s+(i|install|add)\b
  • 実行前にスキャン → スコア集計
  • 結果に応じて hook 出力：
    • permission: "allow" … 安全
    • permission: "ask" … user_message に詳細表示し承認待ち
    • permission: "deny" … 危険な検知でブロック
  • failClosed: true でスキャンが落ちたら止める（安全側）

  機能
依存関係に対する防御の徹底
safenpm は、サンドボックスの執行、静的分析、深層インテリジェンススキャンといった複数のセキュリティチェックを層状に配置し、単一のツールが見逃す脅威を捕捉します。
サンドボックス執行
インストール後のすべてのスクリプトは、ネットワークアクセスがないOSレベルのサンドボックス内で実行されます。悪意のあるパッケージは、電話で帰宅したり、トークンを外部に配布したり、ペイロードをダウンロードしたりできません。
静的解析
19 の検出ルールは実行前にスクリプトをスキャンします。curl/wget のパイピング、DNS のエクスフィル、env の盗用、Base64 の難読化、SSH キーへのアクセスを、重大度加重スコアリングで捕捉します。
タイプスクワット検出
Levenshtein の距離と 5 つの攻撃パターン検出器（char‐swap、置換、スコープの混乱）により、80 以上の一般的な npm パッケージに対してすべての依存関係をチェックします。
行動的な差
キャッシュはプロジェクトごとにスクリプトをインストールします。アップグレード時に、キャッシュされたバージョンと比較して、新たな脅威や新しい警告、リスクスコアの変更が表示されます。
メンテナー変更検出
出版社の履歴について npm レジストリに問い合わせます。パッケージを公開する人物がバージョン間で変更される際に通知し、アカウント乗っ取りのシグナルとなります。
ロックファイル整合性監査
package-lock.json を解析して、git の依存関係、カスタムレジストリ、欠落しているインテグリティハッシュ、弱いアルゴリズム、そして古くなったロックファイルを捕捉します。
評判スコアリング
メンテナー数、ライセンス、リポジトリの有無、依存関係の重み、成熟度に基づき、各パッケージを0〜100点にスコア付けします。プロジェクトの健康等級に集約されます。
分散型脅威ネットワーク
パッケージがブロックされると、シグナルは匿名でsafenpmネットワークに報告されます。インストールごとにそれを問い合わせます — 開発者が1人攻撃された場合、全員に警告が向けられます。識別データは決してあなたの機械から離れません。
CI/CD 準備完了
フルJSON出力モード、ブロック上の非ゼロのエグジットコード、オートローテーション付き監査ログ、そしてパイプライン統合用のスタンドアロンスキャンコマンド。
医師の健康報告書
単一のコマンドで、プロジェクトに文字評価（A+からF）を付与し、6つのカテゴリにわたる加重スコアを付与します。すべての発見には、実行可能な解決策の提案が伴います。
ワンクリック自動修正
タイプスクワットを検出しましたか？safenpm の修正は、疑わしいものを自動的に削除し、正当なパッケージをインストールします。適用する前に、--dry-run でプレビューしてください。
パッケージ差分ビューア
Git-diffスタイルの出力で、各依存関係の更新で何が正確に変更されたかを示します — スクリプトの変更、追加/削除された deps、そして新しいファイル。スナップショットと比較。

Deep Scan

The --scan flag activates all six intelligence modules for a comprehensive security audit of your dependency tree.

# During install:
safenpm install --scan

# Standalone (no install, analysis only):
safenpm scan

# JSON output for CI:
safenpm scan --json
What gets checked

Typosquat detection checks every package name against 80+ popular npm packages using Levenshtein distance and 5 attack pattern detectors including char-swap, substitution (0→o, rn→m), scope confusion, and edit distance analysis.

Behavioral diffing compares install scripts against a cached baseline from your last install. If a package upgrade introduces new curl calls, DNS lookups, or other suspicious patterns, you'll see the exact new warnings and risk delta.

Maintainer change detection queries the npm registry for the publisher of each version. If the account publishing the current version differs from who published the previous one, that's flagged as a potential account takeover.

Lockfile integrity audit parses package-lock.json and checks for git dependencies, custom registries, missing integrity hashes, weak hash algorithms, and stale lockfiles.

Reputation scoring evaluates every package based on maintainer count, license, repository presence, dependency weight, install scripts, and version maturity, producing a 0-100 score and a project-level health grade.

Community threat intel runs automatically on every install — not just with --scan. Each time you run safenpm install, packages are checked against the live decentralized intelligence network, surfacing flagged packages with report counts and attack categories before any scripts execute.
